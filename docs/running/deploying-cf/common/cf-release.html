Using the latest CF-Release

Introduction

CF-Release is the BOSH release repository for Cloud Foundry. Use this with a custom manifest for your environment to deploy Cloud Foundry.

This short guide shows how to, after bootstrapping Bosh, create a Cloud Foundry release ready to deploy to your environment.

Clone CF-Release

Create / find a folder to keep your clone of the CF-Release repository and clone it from Github;

$ mkdir -p ~/src/cloudfoundry && cd ~/src/cloudfoundry
$ git clone -b release-candidate git://github.com/cloudfoundry/cf-release.git && cd cf-release

Update the release

Run the included update script, ~/src/cloudfoundry/cf-release/update;

$ ./update

This is a very simple script that just runs a 'git pull' on the repository and then updates all the submodules. CF-Release is a combination of various other repositories.

Build the release

At this point if the Bosh CLI is not installed on the local system and targeted at a BOSH / MicroBOSH instance, do so, if necessary use this install guide.

When prompted, name your release appcloud:
